Название:Тестирование приложений IOS & Android
Описание:Чек лист для тестироваания приложений IOS $ Android
Окружение:IOS, Android
Автор:Святослав Э.

1.Функциональное тестирование
1.1 Тип приложения соответствует заявленному в требованиях
1.2 Целевая аудитория приложения соответствует требованиям
1.3 Все ли обязательные обязательные поля работают должным образом
1.4 Обязательные поля отображаются на экране отличным образом от необязательных полей
1.5 Переходит ли приложение в свернутый режим при входящем телефонном звонке
1.6 Может ли телефон хранить, обрабатывать и получать SMS при  запущенном приложении
1.7 Возможность переключения на другое приложение
1.8 Приложение поддерживает необходимые параметры социальных сетей, такие как совместное использование, публикация и 
навигация и т.д.
1.9 Пользователь получает сообщение об ошибке, например «Ошибка сети. Пожалуйста, попробуйте через некоторое время» 
всякий раз, когда возникает какая-либо сетевая ошибка
1.10 Работает ли приложение в соответствии с требованиями во всех версиях сетей, то есть 2g, 3g, 4g и 5g
1.11 Gредоставляет ли приложение доступное руководство пользователя для тех, кто не знаком с приложением

2. UI тестирование
2.1 Элементы интерфейса должны быть такого размера, что бы пользовтелю было удобно попадать по ним.
2.2 Отсутствие пустых экранов в приложении
2.3 Многократное быстрое нажатие на кнопку 
2.4 Нажатие на несколько кнопок одноврменно
2.5 Все шрифты соответствуют требованиям
2.6 Все тексты правильно выровнены
2.7 Все сообщения об ошибках верные, без орфографических и грамматических ошибок
2.8 Неактивные элементы отображаются серым
2.9 Анимация между переходами
2.10 Поддерживаются основные жесты при работе с сенсорными экранами (swipe back и т.д.)
2.11 В поисковых строках присутствуют плейсхолдеры

3.Тестирование установки
3.1 Установка приложения занимает разумный промежуток времени
3.2 Удаление приложения занимает разумный промежуток времени
3.3 Готовности серверов к массовому обновлению приложения юзерами
3.4 Приложение устанавливается/обновляется/удаляется через магазин приложений
3.5 Установка/обновление приложения из файла
3.6 Откат к старой версии приложения
3.7 Установка на старую версию приложения

4.Локализации
4.1 Все элементы в приложении переведены на соответствующий язык
4.2 Тексты зависят от языка в системных настройках
4.3 Тексты приходят с сервера
4.4 Корректное отображение форматов дат (ГОД — МЕСЯЦ — ДЕНЬ или ДЕНЬ — МЕСЯЦ — ГОД.)
4.5 Корректное отображение времени в зависимости от часового пояса

5.Стрессовое тестирование
5.1 Высокая загрузка центрального процессора
5.2 Нехватка памяти
5.3 Низкий заряд батареи
5.4 Низкая пропускная способность сети
5.5 Большое количество взаимодействий пользователя с приложением

6. Тестирование безопасности
6.1 Тестирование разрешений (доступ к камере/микрофону/галерее/и т.д.)
6.2 Данные пользователя (пароли) не передаются в открытом виде
6.3 В полях, с вводом пароля и подтверждением пароля, данные скрываются астерисками
6.4 Уязвимости к SQL injection.
6.5  Не разрешает ли приложение злоумышленнику получить доступ к конфиденциальному контенту или функциям без надлежащей 
аутентификации
6.6 Предотвращения небезопасного хранения данных в кеше клавиатуры приложений
6.7 Предотвращение переполнения буфера и случаев повреждения памяти
6.8 Включить управление сеансом для предотвращения доступа неавторизованных пользователей к нежелательной информации

7.Тестирование производительности
7.1 Работает ли приложение в соответствии с требованиями при различных условиях нагрузки
7.2 Обеспечивает ли существующая конфигурация клиент-сервер необходимый оптимальный уровень производительности
7.3 Соответствует ли время отклика приложения требованиям
7.4 Убедиться, что расход заряда батареи, утечки памяти, GPS, производительность камеры находятся в пределах 
требований.
7.5 Производительность сети при перемещении с устройством.

8.Тестирование совместимости
8.1 Пользовательский интерфейс приложения соответствует размеру экрана устройства и никакой элемент не 
являются частично невидимыми или недоступными
8.2 Текст в приложении отображается у всех пользователей
8.3 Приложение сворачивается или приостанавливается в случае вызова, а затем, когда вызов останавливается,
приложение возобновляется.

9.Тестирование восстановления
9.1 Восстановление после сбоя банковской транзакции
9.2  Восстановление приложения после непредвиденных сценариев прерывания / сбоя
9.3 Проверке процесса, при котором соединение приостановлено и системе необходимо 
повторно его установить для восстановления данных, непосредственно затронутых приостановленным соединением.
9.4 Полный разряд АКБ
9.5 Перегрев устройства и уход в режим тротлинга

10.Гайдлайны IOS
10.1 Вывод списков на новый экран
10.2 Кнопки подтверждения находятся слева окна
10.3 Названия продуктов Apple должны быть указаны без опечаток
10.4 В приложении не должны упоминаться сторонние платформы
10.5 Приложение не может использовать устаревшие интерфейсы
10.6 В приложении должна быть хоть какая-то функциональность.
10.7 Функциональность должна соответствовать описанию в магазине приложений.
10.8 Иконки приложения должны соответствовать и подходить ему.
10.9 Вся функциональность должна работать, как описано
10.10 Приложение не может ограничивать пользователей в выборе геолокации или
мобильного оператора
10.11 Приложение не может рассылать спам, распространять вирусы, или использовать 
другие платформы Apple (например, Game Center/Push Notifications) с этими целями.
10.12 Приложение не может использовать геолокационные службы без разрешения.
10.13  Все ссылки в коде приложения должны работать.
10.14 Уведомления должны рассылаться через Apple Push Notification API, при помощи APN ID
10.15 Уведомления не должны содержать личных данных.
10.16 Приложение не может запрашивать личные данные (например, email) пользователя, чтобы
функционировать.
10.17 Если приложение использует сеть, оно не пользуется ей через сторонние (не-Apple)
браузеры.
10.18 Приложение может обновляться только через магазин приложений.
10.19 Приложение не может перепрограммировать контрольные клавиши устройства, не
предназначенные для подобного использования
10.20 Механизм встроенных покупок не может использоваться для приобретения товаров и
услуг, использующихся вне приложения.

11. Гайдлайны Android
11.1 Приложение поддерживает стандартную навигацию по кнопке «Назад» и не использует никаких 
настраиваемых экранных подсказок «Кнопка возврата»
11.2 Уведомления не используются для рекламы другого продукта
11.3 Приложение использует весь экран в обеих ориентациях
11.4 Содержимое приложения и все веб-содержимое, на которое ссылается приложение, поддерживают 
темную тему
11.5 Звук возобновляется, когда приложение возвращается на передний план, или указывает пользователю, 
что воспроизведение находится в приостановленном состоянии
11.6 Если воспроизведение звука является основной функцией, приложение должно поддерживать фоновое воспроизведение
11.7 Приложение загружается быстро или предоставляет пользователю обратную связь на экране (индикатор выполнения), 
если приложение загружается дольше двух секунд.
11.8 Приложения должны отображать кадры каждые 16 мс, чтобы достичь 60 кадров в секунду. 
11.9 Приложение работает на последней общедоступной версии платформы Android без сбоев и серьезного воздействия 
на основные функции.
11.10 Приложение предназначено для последней версии Android SDK, необходимой для соответствия требованиям Google Play
11.11 Приложение правильно поддерживает функции управления питанием, представленные в Android 6.0 (Doze и App Standby)
11.12 Приложение запрашивает только абсолютное минимальное количество разрешений, необходимых для поддержки 
рассматриваемого варианта использования.
11.13 Приложение четко сообщает, почему необходимы определенные разрешения
11.14 Приложение не должно полностью препятствовать доступу пользователя к приложению при отказе в разрешениях.
11.15 Скриншоты и видео приложения не отображают и не ссылаются на устройства, отличные от Android.
11.16 Весь сетевой трафик отправляется через SSL
11.17 Приложение объявляет конфигурацию сетевой безопасности
11.18 Приложение не использует не сбрасываемые аппаратные идентификаторы , такие как IMEI, для идентификации
11.19 Если приложение воспроизводит звук в фоновом режиме, оно должно создать уведомление в стиле MediaStyle
11.20 Если приложение воспроизводит видео, оно должно поддерживать воспроизведение картинки в картинке
